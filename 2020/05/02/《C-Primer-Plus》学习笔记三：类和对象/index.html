发表于 2020-05-02 | 分类于 C++ | 阅读次数:

1，一个完整的类定义

  • 访问控制：默认为private。一般将成员变量设为私有，而将成员函数设为公有，作为外部程序接口。

  • C++结构可以包含成员数据和函数，类和结构的唯一区别在于，结构的默认访问类型是public，而类为private。一般用类实现类描述，而将结构限制为只表示纯粹的数据对象。

  • 类的每个新对象都有自己的存储空间，用于存储其内部变量和类成员；但同一个类中所有对象共享一组类方法，即每种方法只有一个副本。

  • 多文件程序组织：一般将类定义放在一个头文件，然后另一个文件给出类实现，另起一个文件为程序文件。

  • const成员函数：在声明及定义的函数头后加上”:const”，如上述show()方法。可以保证函数不会修改对象。并且可以使用const对象调用该const方法。（const对象不能调用非const方法）

  • 内联方法：有两种方式使得方法为内联方法。

    ​ 1）将内联定义放在定义类的头文件中。如上述代码中的方法set_tot()。

    ​ 2）也可以在类声明之外定义成员函数，并且使用inline限定符使之成为内联。

    ​ 内联方法的规则要求每个使用内联方法的文件中都对其定义，因此简便的做法是将内联定义放在定义类的头文件中。

2，析构函数和构造函数

构造函数在创建类对象时被自动调用。有两种构造函数：

1）默认构造函数

当函数定义未提供构造函数时，C++自动提供类构造函数。但是，如果程序提供了构造函数，C++将不会提供默认构造函数，此时想使用默认构造函数，需要显示定义默认构造函数。

除了解决冲突外，还提高作用域内枚举的类型安全：

  • 有些情况下，常规枚举量将自动转换为整型。
  • 作用域内枚举不能隐式地转换为整型，但可以强制类型转换。

  • 常规枚举，C++11的底层实现随系统而异，但是可以指定底层实现。
  • 作用域内枚举，C++11默认为int，也可指定。如添加:short指定底层为short。
