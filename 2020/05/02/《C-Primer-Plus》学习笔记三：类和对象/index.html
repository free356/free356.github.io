1，一个完整的类定义

  • 访问控制：默认为private。一般将成员变量设为私有，而将成员函数设为公有，作为外部程序接口。

  • C++结构可以包含成员数据和函数，类和结构的唯一区别在于，结构的默认访问类型是public，而类为private。一般用类实现类描述，而将结构限制为只表示纯粹的数据对象。

  • 类的每个新对象都有自己的存储空间，用于存储其内部变量和类成员；但同一个类中所有对象共享一组类方法，即每种方法只有一个副本。

  • 多文件程序组织：一般将类定义放在一个头文件，然后另一个文件给出类实现，另起一个文件为程序文件。

  • const成员函数：在声明及定义的函数头后加上”:const”，如上述show()方法。可以保证函数不会修改对象。并且可以使用const对象调用该const方法。（const对象不能调用非const方法）

  • 内联方法：有两种方式使得方法为内联方法。

    ​ 1）将内联定义放在定义类的头文件中。如上述代码中的方法set_tot()。

5，类中运算符重载

C++扩展了运算符重载，允许定义运算符函数，用来描述特定的运算符与类之间的关系。运算符可以是类成员函数，也可以是友元函数。

如果要为类重载运算符，当运算符函数是成员函数时，则第一个操作数将是调用该函数的对象。

如果需将非类的项作为其第一个操作数，则可以用友元函数来反转操作数顺序。也可以用非友元、非成员函数，此时需要在普通函数中调用成员函数。

有一些运算符函数只能是类成员函数，如下：

  • =：赋值运算符。
  • ()：函数调用运算符。
  • []：下标运算符。
  • ->：通过指针访问类成员的运算符。

6，类的自动转换和强制类型转换

1）将其他类型值转换为类对象

在C++中，任何接受一个参数的构造函数（或者为其余的参数提供了默认值），都可以被用来将与参数相同类型的值转换为类。

隐式调用Stonewt(double)将参数类型的值转换为对象的情况有以下几种：

  • 将Stonewt对象初始化为double的值。
  • 将double值赋值给Stonewt对象。
  • 将double值传递给接受Stonewt参数的函数。
  • 在上述任何一种情况下，使用可转换为double类型的内置类型时。（比如将int转为double，再将double转为Stonewt对象）。

如果使用关键字explicit限定了这种构造函数，则它只能用于显示转换。

2）将类对象转换为其他类型

如果要将对象转换为其他类型，必须定义转换函数，且转换函数必须是成员函数。如 operator typeName();在将类对象赋值给typeName变量时或者将其强制转换为typeName类型时，将自动调用该转换函数。但是当有二义性时，将出错。

应谨慎使用隐式转换函数，通常，最好选择显式转换。避免隐式转换方法有二：

方法一：将转换运算符声明为显示的。

方法二：用一个功能相同的非转换函数替换该转换函数，但仅在被显式地调用时，该函数才会执行。

7，构造函数中动态分配内存

C++自动为每一个类提供了以下成员函数：

  • 默认构造函数，如果没有定义构造函数。（带参数的构造函数，如果所有的参数都有默认值，也是默认构造函数，注意，只能有一个默认构造函数）。
  • 默认析构函数，如果没有定义。
  • 复制构造函数（浅复制），如果没有定义。
  • 赋值运算符（浅复制），如果没有定义。
  • 地址运算符（返回this），如果没有定义。

在类构造函数中，使用new为数据动态分配内存，然后将内存地址赋给类成员，这样，类可以处理长度不同的字符串，而不用提前固定字符串长度。

如果类中包含了使用new初始化、指向数据的指针，而不是数据本身，则需要：

  • 重新定义默认构造函数，在类构造函数中使用new(new [])，
  • 重新定义析构函数，并且需在析构函数中使用和构造函数对应的delete(delete[])。

  • new和delete必须兼容，如果有多个构造函数，则必须以相同的方式使用new，要么都带[]，要么都不带[]，因为只有一个析构函数，所有的构造函数需和析构函数兼容。

  • 则需定义一个复制构造函数，执行深度复制，应分配空间存储并复制指向的数据，而不是指针（默认的复制构造函数执行的是浅复制）以及更新所有受影响的静态类成员。（如果类中包含静态成员，其值需在新对象创建时发生变化，则也需显式提供复制构造函数）。

  • 需重载赋值运算符函数，在其中执行深度复制（默认的是浅复制）具体地，检查自我赋值，释放成员指针以前指向的内存，复制数据而不仅仅是数据的地址，并且返回一个指向调用对象的引用。
  • 如果不提供复制构造函数和重载赋值运算符函数，则需要将所需的复制构造函数和赋值运算定义为私有，这样：第一避免了自动生成的默认方法，第二因为私有，所以不能广泛使用。但是这样做使得对象不可以被复制，并且对象不可以按值传递或返回，需要按照引用传递。并且不可以创建临时对象。（按值传递或者返回对象，或者创建临时对象，都会调用复制构造函数）。

如果类中包含的成员使用了动态内存分配，此时该类并不需要重新编写复制构造函数和赋值运算符，因为默认的逐成员复制和赋值行为有一定智能，逐成员复制和赋值将使用成员类型定义的复制构造函数和赋值运算符。但是，如果Magzine类因其他成员需要定义复制构造函数和赋值运算符时，则需要显式调用Stringuser的复制构造函数和赋值运算符。

复制构造函数

复制构造函数用于将一个对象复制到新创建的对象中，每当程序生成对象副本，都会调用复制构造函数，比如当函数按值传递对象时，或者函数返回对象时。

8，有关函数返回对象

1）返回指向const对象的引用

  • 返回对象将调用复制构造函数，而返回引用不会。
  • 引用指向的对象应在调用函数执行时存在，即不能返回临时对象的引用。
  • v1和v2都被声明为const引用，因此返回类型必须为const，这样才匹配。

2）返回指向非const对象的引用

两种常见的返回非const对象的情况：

  • 重载赋值运算符：用于提高效率（引用）和连续赋值（非const）。
  • 重载与cout一起使用的<<运算符：返回类型必须是ostream&，因为如果返回ostream，则需调用ostream的复制构造函数，而ostream没有公有的复制构造函数。

3）返回对象

如果函数返回的是临时对象，则不能返回引用。函数返回的临时对象可以被修改重新赋值，但这是无意义的（返回对象调用复制构造函数）。

4）返回const对象

防止函数返回来的对象被重新赋值或修改等错误操作，优先返回const对象（返回对象调用复制构造函数）。

9，定位new创建对象与new创建对象

  • 在创建第二个对象时，定位new使用一个新对象来覆盖用于第一个对象的内存单元，因此需要提供不同的地址。
  • 使用定位new运算符为对象分配内存，需显式调用其析构函数。（delete可与常规new运算符配合使用，但不能与定位new配合使用）。（只调用delete[] buffer，将会释放常规new分配的内存块，但它没有为定位new在该内存块中调用的对象调析构函数。

10，成员初始化列表

成员初始化列表允许在对象创建时就对其进行赋值，此时构造函数体语句还没有执行。

当数据成员是非静态const成员或者引用，则必须采用这种方式。（const成员必须在指向到构造体之前，即创建对象时进行初始化，因为const成员以及引用只能在创建时进行初始化，不能赋值）。

只有构造函数可以使用这种初始化列表语法。但是成员初始化列表不限于初始化常量，其他成员也可以使用。

一种等价的方法是在类声明的时候就进行初始化，但是使用成员初始化列表的构造函数将覆盖相应的类内初始化。

数据成员被初始化的顺序与它们出现在类声明中的顺序相同，与初始化器中的顺序无关。

11，类继承

1）简单继承

派生是一个is-a关系，派生类继承基类的数据成员和大部分方法。但不继承基类的构造函数、析构函数和赋值运算符。

当需要派生类时，需要定义自己的构造函数：派生类的构造函数需通过成员初始化列表显式定义基类的构造函数，如果没有显式调用基类构造函数，将自动调用基类的默认构造函数。

因为创建派生类对象时，应首先创建基类对象，即基类对象应在程序进入派生类构造函数之前被创建，因此，需要使用成员初始化列表显式调用基类构造函数。

如果不通过成员初始化列表，将会先调用基类默认构造函数，再通过赋值运算符设置派生类中的基类数据成员。

基类和派生类的关系：

  • 派生类对象可以使用基类的非私有方法；

  • 基类指针或引用可以在不进行显式类型转化的情况下指向或引用派生类对象，如Base *pr=&son;但是，基类指针或引用只能调用基类方法，不能通过其调用派生类的方法。

  • 也可以用派生类初始化基类或者将派生类赋值给基类；

    派生类与基类相互赋值：

    1）将派生类对象赋给基类对象

    将派生类对象赋给基类对象，将调用左边对象的赋值运算，即基类的赋值运算。

    2）将基类对象赋值给派生类对象

    一般情况下，不允许将基类对象赋值给派生类对象，除非以下三种情况：

    • 如果派生类包含了这样的构造函数，即对基类对象转换为派生类对象进行了定义。
    • 如果派生类定义了用于将基类对象赋值给派生类对象的赋值运算符。
    • 如果发生了强制类型转换。

2）多态公有继承

如果要在派生类中重新定义基类的方法，通常应将基类方法声明为虚的。此时，程序将根据引用或者指针指向的对象的类型来选择方法，如果没有将基类方法声明为虚的，程序将根据引用类型或指针类型选择方法。从而实现多态。

方法在基类中声明为虚的，它将在派生类中自动成为虚方法。

析构函数最好是虚的，这样可以确保以正确的顺序调用析构函数。

在派生类中调用基类的方法，如果该方法没有在派生类中重新定义，将不需要使用作用域解析符；如果该方法在派生类中重新定义了，将需要使用作用域解析符调用基类中的该方法。

3）派生类重新定义基类的方法

在派生类中重新定义基类的方法，遵循两个原则：

  • 如果重新定义继承的方法，应确保与原来的原型完全相同，但如果返回类型是基类引用或指针，可以修改为指向派生类的引用或指针。

  • 如果基类声明被重载了，则需在派生类中重新定义所有的基类版本。（如果只重新定义基类的一个版本，则基类其他版本将会被隐藏）。

12，抽象基类

当两个类定义为继承不太合适，但是又具有很多共性时，可以抽取出共性，将其定义为抽象基类。

抽象基类描述必须至少包含一个纯虚方法（在声明时使得方法=0）的接口。不一定非得定义纯虚方法（在实现中定义）。

13，继承与动态内存分配

当基类使用动态内存，派生类没有使用动态内存时，不需要为派生类显式定义析构函数、复制构造函数和赋值运算符。

当基类和派生类都使用动态内存时，需要为派生类显式定义析构函数、复制构造函数和赋值运算符：

  • 派生类析构函数中delete掉派生类新增成员，基类析构函数将会自动调用。

  • 复制构造函数显式调用基类的复制构造函数，通过成员初始化列表将子类的引用传递给基类的复制构造函数。

  • 赋值运算符通过作用域运算符显式调用基类的赋值运算符。

14，补充几个小内容点

1）有关类中嵌套定义结构

C++允许在类中声明结构、类和枚举定义，这些嵌套类型的作用域为整个类。这种声明不会创建数据对象，只是指定类可以使用的数据类型。如果声明在类的私有部分，则只能在这个类中使用被声明的类型；如果声明是在公有部分，则可以在类的外部通过作用域解析符使用被声明的类。这意味着它们被局限在类中，不会与其他地方定义的同名结构、类和枚举发生冲突。

2）静态成员函数

静态成员函数不能通过对象调用，也不能使用this指针。并且只能使用静态数据成员。

3）派生类使用基类的友元

派生类使用基类的友元，须通过强制类型转换。
